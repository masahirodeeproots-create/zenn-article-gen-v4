# Style Guide

## IMPORTANT Rules
<!-- 最大15個。超過時は退役処理 -->

- [voice] です・ます体を基本とし、「だ/である」「体言止め」は禁止。カジュアル表現（「〜なんですよね」「まあ正直」）は自然に混ぜる
  - 追加日: 2026-04-08
  - 根拠: 「だ/である」混在はAI臭の最大要因

- [rhythm] 3文以上同じ文長（短/中/長）が続かないようにする。短→長→短の波を意識
  - 追加日: 2026-04-10
  - 根拠: 同じリズムが続くと読者が離脱する（ART-001連続同長文帯として4iteration継続中）

- [style] HARD FAIL: です・ます比率≥80%を確保するための文末チェック（今回の学習）
  - 同形文末「〜しました」「〜していました」が4文以上連続する箇所を特定し、1〜2文を長文または異なる文末形式（「〜しています」「〜になります」）に変換する
  - 箇条書き行末は体言止め・辞書形禁止。必ず「〜しました」「〜しています」形式で統一する
  - KOL出力例・絵文字前の文末が辞書形になりやすい。絵文字直前でも文末をです・ます体で終わらせる
  - 追加日: 2026-04-14
  - 根拠: 67.9%（目標80%まで12pt不足）。幕1の4エージェント行動列挙が集中ポイント（今回の学習）

- [style] HARD FAIL: だ/である体の混入しやすい箇所パターン（iteration 2の学習）
  - シミュレーション対話内のClaude発言文末: 「〜があって。」→「〜があります。」
  - コードブロック内のコメント文末: 「// 〜する」→「// 〜します」
  - KOL出力例の末尾: 絵文字直前でも文末をです・ます体で終わらせる（辞書形・体言止め禁止）
  - 追加日: 2026-04-14
  - 根拠: 71.1%（iteration 2）。上記3箇所が未修正だとです・ます比率カウントが漏れ続ける（iteration 2の学習）

- [style] HARD FAIL: 間接疑問形連続（〜か。〜か。〜か。）は辞書形文末として計上されです・ます比率を引き下げる（iteration 3の学習）
  - 「どのXがYをするか。どのZがWか。」の形の間接疑問文3連続は散文1文に統合する
  - 例: 「どのplatformがどのスライドタイプを使うか。どのShapeにどのasset_idを割り当てるか。」→「どのplatformがどのスライドタイプを使い、どのShapeにどのasset_idを割り当てるか——これらすべての選択が、複数ファイルに分散したルールの解釈に委ねられていました。」
  - 間接疑問形は1文につき1個まで。2個以上連続する場合は連結または散文変換する
  - 追加日: 2026-04-14
  - 根拠: 70.6%（iteration 3）。line 63の間接疑問形3連続が前iteration 2から0.5pt悪化の主因（iteration 3の学習）

- [style] HARD FAIL: 技術説明文内の動詞辞書形文末「〜を入れる。」「〜を始める。」パターン（iteration 4の学習）
  - コードブロック直後の補足説明や操作手順説明文で、動詞が辞書形のまま文末に来る
  - 「`Shape`には`asset_id`を入れる。」→「入れます。」、「作業を始める。」→「始めます。」
  - 修正後は前後2文も確認し、連鎖的な辞書形文末がないことを確認する
  - 追加日: 2026-04-14
  - 根拠: 71.6%（iteration 4）。line 129「入れる。」×2・line 151「始める。」が間接疑問形解消分の改善を相殺した（iteration 4の学習）

## Learned Rules

- [rhythm] セクション冒頭は短文で入る（15字以内）
- [voice] 一人称で書くとき「私は」を連発しない。省略で十分
- [structure] 技術的な説明は「結論→理由→具体例」の順序
- [distance] 他ドメインとの類推（「〜と同じ」）は1記事2-3個まで。7つ並べると「借り物の知識」感
- [rhythm] 段落接続を多様化。「で、」「んで、」だけで全段落をつなげない
- [density] セクション行数比は最長:最短で3:1以上を維持
- [structure] 連番見出し（Round N）を使うなら全要素に見出しを。崩れるなら連番構造を立ち上げない
- [voice] フロントマターは title, emoji, type, topics, published を含める
- [style] 箇条書き行末もです・ます体で統一。体言止め・辞書形は箇条書き内でも禁止（今回の学習）
- [structure] 複数エージェントの行動を並列列挙するとき、1エージェントの視点に絞り1段落を散文描写に変換する（今回の学習）
- [structure] 3幕の構造スタイルを意図的に差別化する: 幕1=問題を未解決のまま次へ渡す型、幕2=結論から逆算型、幕3=複数視点交差型（今回の学習）
- [structure] 同一構造の列挙節が2節以上隣接する場合は片方を時系列展開または散文変換に切り替える（iteration 2の学習）
- [structure] 冒頭2段構成：「何の話か」→「フック」。3文以内。フックが普遍的なら先に出す（IMPORTANT卒業: iter 2-4で違反なし）
- [density] 記事の最も強い数字（時間削減・コスト・達成率）は冒頭フックか冒頭セクション内に出す（IMPORTANT卒業: iter 2-4で違反なし）
- [structure] 記事全体を貫く「1つの概念的フレーム」を冒頭で明示。テーゼは1つ（IMPORTANT卒業: iter 2-4で違反なし）
- [distance] セクション冒頭は出来事・対話・エラーメッセージ・具体的操作で開始。ナレーション宣言禁止（IMPORTANT卒業: iter 2-4で違反なし）
- [structure] 全セクション等間隔「失敗→原因→解決策」テンプレート禁止。少なくとも1セクションは結論から逆算、1セクションは問題を未解決のまま次へ渡す（IMPORTANT卒業: iter 2-4で違反なし）
- [density] コードブロック後に必ず1-2文の解説を入れる。コード比率20%超は「貼っただけ」感（IMPORTANT卒業: iter 2-4で違反なし）
- [distance] 読者への問いかけは1セクション1回まで。「なんか面白くないですか？」連発禁止（IMPORTANT卒業: iter 2-4で違反なし）
- [emotion] 失敗談のあとにユーモアを1つ入れると緩和される（IMPORTANT卒業: iter 2-4で違反なし）
- [voice] 専門用語は初出時に1文で説明する。括弧注釈は1記事2箇所以内（IMPORTANT卒業: iter 2-4で違反なし）
- [voice] 記事全体を貫く概念フレームの語は冒頭3段落以内に登場させる。技術用語と非技術メタファーを交互に使い、まとめでもフレームの語を呼び出す（IMPORTANT卒業: iter 2-4で違反なし）
- [rhythm] 連続同長文帯が4iteration連続で未解消の場合、文末修正では解消できない。帯の中に「ほぼ勘です。」のような明示的短文を1文差し込み分断する（iteration 4確認）

## Failure Patterns

- [rhythm] 同じ文長の文が4文以上続くと読者が離脱する
- [density] コード比率20%超で「コード貼っただけ」感が出る
- [structure] スペック羅列で冒頭を開始すると読者は「それ何？」と置いてけぼり
- [distance] 普遍的フックより先にシステム説明を置くと没入が1-2文遅延する
- [rhythm] セクション冒頭のナレーション宣言は体験記ベンチマークに存在しない
- [structure] 全セクション等間隔テンプレートは後半でリズムが読めてしまう
- [style] 同形文末「〜しました」の短文連打（4文以上）はです・ます比率を急落させ、かつリズムを単調にする二重の問題を引き起こす（今回の学習）
- [structure] 3幕全てが「失敗→解決策」構造になると幕3の構造が読める。幕3は「変化後の現実」「未解決の告白」「視点の交差」いずれかで差別化する（今回の学習）
- [rhythm] エージェント行動列挙（「Xが〜します。Yが〜します。」型）が2節以上隣接すると連続同長文帯（7文以上）を形成する。4文を超える前に散文段落か時系列展開で分断する（iteration 2の学習）
- [style] 間接疑問形「〜か。」が3文連続すると辞書形文末3件分としてです・ます比率を急落させる。列挙的な「〜か。〜か。〜か。」構文は書いた直後に1文統合を強制する（iteration 3の学習）
- [rhythm] 連続同長文帯（閾値4文）が3iteration連続で解消されない場合、問題は文末修正ではなく構造（列挙節の隣接配置）にある。節単位の構造変換（散文化・時系列展開）で対処する（iteration 3の学習）
- [style] 技術説明文（「〜にXを入れる。」「作業を始める。」型）は辞書形動詞が文末に来ても違和感が薄く見落としやすい。前の問題を修正した直後に、隣接文（前後2文）も確認する（iteration 4の学習）
