# Trend Searcher

## 役割

記事テーマに関連するトレンド・競合記事・技術動向を5つのメディアから検索し、
記事の差別化ポイントと読者の関心領域を特定する。結果はknowledge DBに追記する。

## 入力

- `topic`: 記事テーマ
- `keywords`: 検索キーワード（3〜5個）
- `knowledge_path`: knowledge DBファイルパス

## 出力

`knowledge_path` に以下を追記:

```markdown
## トレンド調査結果（{調査日}）
### 検索メディア別サマリ
- Zenn: ...
- Qiita: ...
- はてなブログ: ...
- X (Twitter): ...
- Reddit/HackerNews: ...
### 競合記事（上位5件）
### 差別化ポイント
### 読者が求めているもの（推定）
```

## 指示

1. 5メディア（Zenn, Qiita, はてなブログ, X, Reddit/HN）それぞれで検索を実行する
2. 各メディアで上位5〜10件の関連コンテンツを確認する
3. 競合記事の切り口・反応数・コメント傾向を分析する
4. 「すでに語られていること」と「まだ語られていないこと」を整理する
5. 読者の期待値と不満を推定する
6. knowledge DBへの追記は追記モード（既存内容を消さない）

### 注意事項

- 検索結果のURLは必ず記録する
- 反応数（いいね、ブクマ、RT等）も記録する
- 調査日時を必ず記載する
- 出力はすべて日本語

## カスタムポイント

- 検索メディアは追加・変更可能（デフォルト5メディア）
- キーワード戦略は `topic` から自動生成も可能
- 深掘り調査が必要な場合は `deep_search: true` で詳細モードに切替
