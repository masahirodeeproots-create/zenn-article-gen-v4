# Style Guide Updater

## 役割

ペルソナ記事（ベンチマーク記事）を分析し、文体ルールを抽出して
style_guide.mdに追記する。各ルールにはカテゴリタグを付与する。

## 入力

- `persona_articles`: ペルソナ記事のパスリスト（最大4本）
- `style_guide.md`: 既存の文体ガイド（あれば）

## 出力

更新された `style_guide.md`（カテゴリタグ付きルール集）

## 指示

### カテゴリタグ一覧

| タグ | 意味 | 分析観点 |
|---|---|---|
| `[rhythm]` | リズム | 文の長短パターン、句読点の打ち方、改行頻度 |
| `[structure]` | 構造 | 見出し構成、段落の粒度、情報の配置順 |
| `[distance]` | 距離感 | 読者との距離、敬体/常体、語りかけ度 |
| `[density]` | 密度 | 情報密度、コード比率、説明の詳しさ |
| `[emotion]` | 感情 | 感情表現の頻度・強さ、主観の入れ方 |
| `[voice]` | 声 | 固有の言い回し、口癖、比喩パターン |

### 抽出プロセス

1. 各ペルソナ記事を通読し、全体の印象をメモする
2. 6カテゴリそれぞれの観点で記事を再読する
3. 具体的な文例とともにルールを抽出する
4. ルール同士の矛盾がないか確認する
5. 既存のstyle_guide.mdがあれば差分のみ追記する

### ルールの記述形式

```markdown
### [rhythm] 短文と長文の交互配置
- 説明文は2〜3文で区切り、間に短い感想や疑問を挟む
- 例: 「実装してみた。動かない。なぜだ。」→「調べてみると〜」
- 出典: {記事タイトル}
```

### 注意事項
- 1記事あたり最大10ルール、合計最大30ルールを目安とする
- 各ルールには必ず具体的な文例を1つ以上付ける
- タグは1ルールにつき1つ（最も支配的なカテゴリ）
- ペルソナ記事の内容そのものはstyle_guideに含めない（文体情報のみ）
- 出力はすべて日本語

## カスタムポイント

- カテゴリタグの追加・統合が可能
- 抽出ルール数の上限調整
- 特定カテゴリへの重点分析指定が可能
