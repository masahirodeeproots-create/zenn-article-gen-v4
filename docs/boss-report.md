# 記事生成ベンチマーク — 状況報告

## やりたかったこと

「固定ベンチマーク → エージェントがフローを自動生成 → フロー同士をスコア比較してプロセスを磨く」

```
固定ベンチマーク（物差し）
  │ 同じ条件で測定
  ▼
フローA → スコア 6.8
フローB → スコア 7.4
フローC → スコア 8.1
  │
  ▼
「Cが一番良い。なぜか？→ 次のフローDに活かす」
```

## 思考の迷走フロー

```
① 固定ベンチマーク作ろう
   ↓
② 「体験記」と「比較検証」で良い記事の定義が全然違う...
   固定の評価軸だと正しく測れないのでは？
   ↓
③ じゃあ記事の方向性に応じて評価軸を動的に生成しよう
   ↓
④ 評価軸を動的に作るなら、エージェント構成やワークフローも
   AutoAgentみたいに動的生成したい
   ↓
⑤ 気づいたら記事生成システム丸ごと作っていた
   肝心の「固定ベンチマーク」が存在しない
```

**②→③の判断で道を逸れた。**
「固定だと測れない」→「じゃあ動的に」は記事を良くする方向としては正しいが、ベンチマーク（=固定の物差し）としては本末転倒だった。

## 次にやること

今あるシステムから「ベンチマーク部分」を固定テストスイートとして切り出す。

1. 固定お題セット（3-5本）を選定
2. 固定の評価基準を1つ作る（今は毎回動的生成している部分を固定化）
3. 「同じお題 × 同じ基準 → スコア」で フロー間比較ができる状態にする
